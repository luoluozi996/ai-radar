WorkOpenAI

OpenAI 展示销售团队如何使用 ChatGPT Work

OpenAI Academy 发布销售团队使用 ChatGPT Work 的案例，信号很直接：AI 工作入口正在从“个人聊天”变成“团队工作流里的任务助理”。

为什么关心
如果你做销售、运营、客户成功或自由职业，ChatGPT Work 的案例可以直接转成自己的流程模板。
今天能做
把一个重复销售任务拆成“资料整理、客户画像、邮件草稿、跟进提醒”四步，试着让 AI 参与其中一环。
观察点
AI 工作区是否会比单个聊天窗口更适合团队协作。
来源：OpenAI Academy
Data WorkOpenAI

OpenAI 展示数据科学团队如何使用 ChatGPT Work

数据科学团队案例说明 AI 不只是写文案，也能进入数据清洗、分析假设、代码草稿和结果解释这类知识工作。

为什么关心
普通人做表格分析、增长复盘、运营报表时，也可以借鉴数据团队的 prompt 和检查方式。
今天能做
拿一份 CSV 或表格，让 AI 先提出 5 个分析问题，再让它解释每个问题需要哪些字段。
观察点
AI 生成分析结论时，是否能同时给出验证方法和不确定性。
来源：OpenAI Academy
DeveloperGitHub

GitHub Copilot for JetBrains 扩展 BYOK 能力

GitHub Changelog 显示 Copilot 在 JetBrains 里扩展 BYOK（Bring Your Own Key）能力，开发团队对模型后端、成本和权限会有更多控制空间。

为什么关心
如果你在公司里用 JetBrains + Copilot，BYOK 会影响模型选择、预算控制和合规配置。
今天能做
检查团队 IDE 插件版本和 Copilot 管理设置，确认是否已经支持新能力。
观察点
AI 编程工具会越来越从“个人插件”变成“团队可治理基础设施”。
来源：GitHub Changelog
SecurityGitHub

GitHub Code Scanning 在 PR 里显示 AI security detections

GitHub Changelog 提到 code scanning 会在 pull request 中显示 AI security detections，这和今天项目页里的 Strix 信号互相呼应。

为什么关心
开发者以后会在 PR 阶段更早看到 AI 辅助的安全提示，而不是等上线后再补安全审查。
今天能做
如果你维护项目，可以检查 code scanning 是否启用，并观察 AI detection 的误报和漏报。
观察点
安全工具正在从“扫描报告”转向“嵌入开发流程”。
来源：GitHub Changelog
