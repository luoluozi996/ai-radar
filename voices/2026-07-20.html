GitHub 的会话 credit 上限观点很朴素：自动化中没有人盯着时，预算约束是工作流的一部分。

争议/限制：软上限可能因正在返回的响应而略超出，不能替代总预算与权限控制。

怎么用：为每个批处理设单次上限与失败后的人工接管点。

把 agent 输出放回代码审查，而不是只看完成标记

性能 PR 研究显示改动类别可被系统性描述；可复用的观点是：用变更类型、测试与回滚策略来审视 agent，而非只信最终分数。

争议/限制：样本聚焦性能优化 PR，不直接等同于所有软件任务。

怎么用：对高风险路径要求 diff 摘要、测试证据和人工批准。
