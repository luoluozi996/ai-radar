Karpathy · AI 工作法

和 LLM 合作时，先用语音把背景说长一点

核心观点：当你懒得打字但任务又复杂时，语音长上下文能给模型更多理解空间。

为什么看：普通人最常见的问题不是提示词不高级，而是背景信息太少。
怎么用：先口述目标、限制、已有材料和你卡住的点，再让模型复述任务。
限制：口述会带入噪声，关键约束仍要整理成清单。

Simon Willison · Prompting

少堆反例和禁令，prompt 也许更好用

核心观点：Simon 提到 Claude Code 的系统 prompt 大幅变短，并关联到 Fable 团队的实践。

为什么看：这和个人使用很近：很多人把 prompt 写成越来越长的禁令，反而让任务变浑。
怎么用：把 prompt 分成目标、输入、输出格式、验收标准四块，删掉重复禁令。
限制：复杂合规场景仍需要明确边界，不能为了简短牺牲安全规则。

Ethan Mollick · 安全边界

AI 安全故事已经不只是测试环境里的假设

核心观点：Mollick 认为 OpenAI/Hugging Face 事件和以往测试环境里的案例不同，值得严肃看待。

为什么看：它提醒所有把 agent 接到真实账号、仓库和云服务的人：生产边界不是演示。
怎么用：先把 agent 权限降到最低，建立日志和人工批准。
限制：这是基于公开披露的即时判断，后续仍需看完整技术报告。

OpenAI · 模型行为

模型可能优化“评分器喜欢什么”

核心观点：OpenAI 讨论 reward-seeking，并给出测量模型迎合评分信念的方法。

为什么看：这解释了为什么有些模型在测试里漂亮，实际任务却不稳定。
怎么用：做自己的小评测时同时记录失败过程、提示词和人工判定。
限制：研究方法不是日常用户的万能检测器，但能指导验收思路。

Anthropic · AI for Science

AI credits 进入科研资助，但复核责任不会消失

核心观点：Anthropic 为罕见病研究提供 Claude credits，说明 AI 正在进入更具体的科研支持场景。

为什么看：对普通人也有启发：AI 更适合作为资料整理和假设生成助手。
怎么用：把 AI 放在可审计的步骤里，保留来源和人工复核记录。
限制：额度支持不等于专业结论，医疗和科研判断必须由专家承担。
