Methodology

日报如何判断“值得看”

AI Content Radar 不追求把所有新闻都搬进来，而是把公开信号转成可判断、可验证、可行动的信息。这个页面说明数据从哪里来、什么会被过滤、条目如何排序，以及自动发布时如何保护密钥与原始数据。

核心原则 原始来源优先，可验证优先，普通使用者相关性优先；对无法核验、只有营销话术或缺少上下文的内容降权。

数据来源

不同来源承担不同角色：有的验证项目是否真实可用，有的补充社区反馈，有的追踪模型与产品变化。

GitHub

用于发现真实项目、README、demo、许可协议、更新活跃度与使用门槛。

X API

用于捕捉 AI 创作者、研究者、产品团队与高质量社区的实时讨论。

HN / 社区讨论

用于观察开发者反馈、反对意见、实际使用成本与早期争议。

RSS / 官方博客

用于追踪产品发布、平台政策、定价、入口与生态变化。

Hugging Face / 模型社区

用于跟踪模型发布、权重开放、评测结果与部署可行性。

论文 / 技术文档

用于验证研究主张、方法细节和能力边界，避免只引用二手解读。

筛选标准

每条内容都会先过一层质量门槛，避免日报变成热搜列表。

必须有可回溯来源

优先保留官网、README、论文、产品文档、原帖或明确讨论链接；无法追溯来源的内容不进入公开页。

必须能回答“对谁有用”

入选条目需要说明它影响开发者、AI 使用者、内容创作者、求职者或产品团队中的哪一类人。

降低纯营销噪音

只有口号、融资、转发热度或模糊愿景，但缺少 demo、证据、用户反馈的内容会被降权。

保留反对意见和使用成本

社区吐槽、失败案例、限制条件、价格和权限变化也会被纳入，因为它们更接近真实使用体验。

评分逻辑

评分不是绝对排名，而是帮助日报在有限篇幅里优先展示更值得复盘的信号。

新鲜度

是否发生在最近时间窗口内，是否仍有后续讨论。

可信度

是否来自原始来源、官方信息、作者实测或多方交叉验证。

实用性

是否能转化为试用、学习、决策、避坑或工作流改进。

影响面

是否影响较多普通使用者，而不只是大公司内部叙事。

可复盘性

是否留下清晰链接、步骤、证据或观点，方便之后回看。

自动化闭环

从抓取到发布的每一步都尽量留下可检查结果，减少手动复制和重复创建。

抓取

按日期窗口读取配置化来源。

去重

用链接、标题、来源和 Duplicate Key 合并重复条目。

筛选

过滤弱来源、低相关性和无行动价值内容。

生成

按栏目渲染静态 HTML，并保留归档入口。

同步

Notion 先查重再更新，避免重复创建日报和项目行。

通知

发布后发送微信推送，并在记忆文件记录脱敏状态。

会发布什么

公开页面只展示筛选后的标题、摘要、判断理由、适用场景与来源链接。它适合阅读、分享和放进作品集。

不会发布什么

不会发布 X Bearer Token、Server 酱 SendKey、原始 API 响应、本地缓存文件、私人配置或任何无法确认授权公开的内容。
