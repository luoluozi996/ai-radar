Hermes Agent

Nous Research 的可自建 agent，README 明确提供终端、消息平台网关、定时自动化、跨会话记忆及多后端执行路径，并给出 Windows 安装方式和完整文档。

适合谁
想让 agent 长期运行、跨 Telegram/Slack 等入口工作，并愿意自己设定工具与权限边界的开发者。
为什么看
它把“技能、记忆、调度、运行环境”整合成完整运行面，而非只提供聊天界面。
限制
记忆与自主技能并不等于可靠；先在容器或低权限环境测试，避免直接授予生产凭据。

nanobot

HKUDS 的轻量个人 AI 助手项目，README 提供中英文文档、PyPI 安装入口和多渠道使用说明，定位是让个人 agent 更容易落地与扩展。

适合谁
希望先低成本体验自托管 agent，又不想从大型框架的全部组件开始的人。
为什么看
“轻量”是实际采用优势：更小的初始复杂度更适合先完成一条可复核的任务链。
限制
轻量不代表功能或安全默认齐备；接入外部工具前仍需检查权限和数据去向。

Cherry Studio

跨 Windows、macOS、Linux 的多模型桌面客户端。README 列出云端与本地模型、文档/PDF、知识管理和 MCP 等入口，适合做个人模型工作台。

适合谁
需要同时调用多个模型、希望保留本地模型与文档工作流选择权的知识工作者。
为什么看
多模型切换正变成常态；它提供一个低代码的比较、整理和连接 MCP 的界面层。
限制
AGPL-3.0 社区版有部署与分发约束；企业用前需先核对许可与数据治理要求。
