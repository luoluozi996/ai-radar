今日模型判断

今天最值得看的是“开权重与可定制化”正在重新成为模型竞争变量。

开权重压力上升

Moonshot、Thinking Machines 等新模型信号说明，企业会越来越关心能否下载、微调、部署和控制成本。

Agent 评测变细

研究侧不再只测聊天能力，而是测长任务、文献发现、真实研究流程和多媒体后期制作。

可用性比榜单更重要

日报后续要记录 demo、权重、API、license、评测限制，而不是只贴模型名和分数。

模型与产品信号

这些条目会影响企业是否选择闭源 API、开权重部署或可定制模型平台。

Open-weightMoonshot Kimi K3 预期发布

FT 报道称 Moonshot 准备推出 Kimi K3，参数规模可能达到 2-3T，并计划以开权重方式挑战 Anthropic 等闭源模型。

看点
中国开权重模型可能继续压低闭源模型价格和部署门槛。
风险
报道为预发布信号，具体能力、license 和实际开放程度仍需等官方确认。
来源：Financial Times
CustomizableThinking Machines 发布首个模型 Inkling

Axios 报道 Mira Murati 创办的 Thinking Machines 发布 Inkling，并强调企业定制、开权重和可微调平台 Tinker。

看点
它不是单纯追求“最聪明模型”，而是把可定制性当作产品卖点。
适合谁
需要用内部数据微调模型、但又不想完全依赖闭源 API 的企业团队。
来源：Axios
Enterprise ModelOpenAI GPT-5.6 与 ChatGPT Work 进入工作场景

Axios 报道 OpenAI 发布 GPT-5.6 和 ChatGPT Work，信号不是单个模型，而是模型能力与企业工作入口继续绑定。

看点
工作场景里，可靠性、权限、协作和流程集成可能比单次 benchmark 更关键。
下一步
观察 ChatGPT Work 是否会和 Codex、文件、组织知识、任务自动化更深整合。
来源：Axios

研究与 Benchmark

这些研究适合放进“长期判断框架”，用来区分模型演示和真实能力。

Agentic Work

The Shift to Agentic AI: Evidence from Codex

基于 Codex 使用数据，观察 agentic AI 如何进入组织工作流，并出现多 agent 并发管理、skills 复用和更复杂任务请求。

可带走：Agent 采用不是“换聊天工具”，而是工作组织方式变化。 arXiv
Research Agent

AutoResearchBench

评估 AI Agent 做复杂科学文献发现的能力，强调 deep research 和 wide research 任务；结果显示强模型在该类任务上仍明显受限。

可带走：研究型 Agent 的瓶颈在检索、判断和证据链，而不只是摘要能力。 arXiv
Real Workflow

AgenticVBench

用真实后期制作任务评估多模态 Agent，覆盖文本、图像、音频、视频理解和长程工具使用。

可带走：行业工作流 benchmark 会比通用问答更能暴露 Agent 的失败模式。 arXiv
