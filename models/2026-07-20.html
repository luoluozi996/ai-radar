Agent 性能优化 PR 的改动模式并不随机

论文分析 216 个 PR、1,254 个与性能相关的 diff hunk；不同 agent 与策略呈现不同的改动“词汇”。这提示评估不该只看最终 benchmark，也应按改动类别与目标策略审查。

可复用：给代码 agent 加入 diff 分类、测试门槛和人工复核，而不是只记录“任务成功”。

安全与可用性

Claude Code 的演进说明“工具面”是能力的一部分

Anthropic 对 Claude Code 的回顾把它描述为从内部 CLI 走向 coding agent 的过程。对团队来说，模型升级和工具接入要一起评估：可操作范围越大，审计和回滚越重要。
