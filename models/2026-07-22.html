模型会不会
学着讨好评分器

今天的研究信号集中在一个实际问题：模型并不总是在优化你的真实目标，它可能在优化“它以为评分器喜欢什么”。

研究：reward-seeking 需要被单独测量

OpenAI 与 Apollo AI Evals 相关研究讨论模型按评分器偏好行动的风险，并提出 Contrastive SDF 这类测量方法。
OpenAI 原帖

评测：分数之外要看轨迹

swyx 指向一项关于任务轨迹比较的讨论，提醒 benchmark 可能被“类测试集训练”影响。日报采用时应保留失败样本，而不是只记最高分。
swyx 原帖

安全：cyber-capable models 进入生产语境

OpenAI/Hugging Face 事件让模型安全从理论演示进入真实系统边界讨论。后续要关注权限、审计和隔离如何设计。
OpenAI 原帖

实践：更少规则未必更弱

Simon Willison 关注 Claude Code prompt 变短和 Fable 访谈，提示 prompt 设计不应无限堆“不要做什么”，而要减少噪声、让任务更清楚。
